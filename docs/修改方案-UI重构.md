# UI重构修改方案

## 项目：中国象棋·开局大师

---

## 一、棋谱选择页（opening 页面）

### 1.1 布局结构

```
┌──────┬──────────────────────────────────────┐
│      │  ┌────┐  中炮对顺手炮            ▶  │
│ 中炮 │  └────┘                               │
│      │  ┌────┐  中炮对屏风马            ▶  │
│ 飞相 │  └────┘                               │
│      │  ┌────┐  中炮对反宫马            ▶  │
│ 仙人 │  └────┘                               │
│      │  ┌────┐  中炮对单提马            ▶  │
│ 起马 │  └────┘                               │
│      │  └────┘                               │
└──────┴──────────────────────────────────────┘
```

- 左侧Tab栏：宽度100px，深色背景(#2d2d2d)，选中项高亮(#8B4513)，白色文字
- 右侧列表：每行高度70rpx，左侧缩略图(80×60rpx占位图)+中间棋谱名+右侧箭头
- 点击棋谱行 → `wx.navigateTo` 跳转游戏页并传递 `openingId`

### 1.2 文件修改

| 文件 | 修改内容 |
|-----|---------|
| `pages/opening/opening.wxml` | 重构为左侧Tab + 右侧图文列表 |
| `pages/opening/opening.wxss` | 新样式 |
| `pages/opening/opening.js` | 简化逻辑：去掉详情/视频/讲解，直接跳转 |
| `data/openings/data.js` | categories加icon字段，openings加image字段 |

### 1.3 资源占位

| 资源 | 路径 | 规格 |
|-----|------|------|
| Tab图标 | `/static/images/tabs/{category-id}.png` | 60×60px |
| 棋谱缩略图 | `/static/images/openings/{opening-id}.png` | 80×60px |

## 二、游戏页（game 页面）

### 2.1 页面结构

```
┌────────────────────────────────────┐
│  ← 返回                            │
├────────────────────────────────────┤
│                                    │
│         内容区域                    │
│    (视频/棋盘+讲解/棋盘+操作)       │
│                                    │
├────────────────────────────────────┤
│  操作按钮区                        │
├────────────────────────────────────┤
│  [看视频]    [讲解]    [练习]      │
└────────────────────────────────────┘
         ↑ 最底部固定
```

### 2.2 三种模式

| 模式 | 说明 | 首次进入 |
|-----|------|---------|
| video | 视频播放 | ✅ 默认 |
| explain | 棋盘+讲解内容+上一步/下一步 | |
| practice | 可交互棋盘+悔棋/重新开始 | |

### 2.3 各模式详细结构

#### 视频模式（默认）
```
┌────────────────────────────────────┐
│         视频播放区域                │
├────────────────────────────────────┤
│  [看视频]    [讲解]    [练习]      │
└────────────────────────────────────┘
```

#### 讲解模式
```
┌────────────────────────────────────┐
│         棋盘区域                   │
├────────────────────────────────────┤
│  棋谱讲解内容区                    │
│  (未开始：显示棋谱简介)             │
│  (进行中：显示每步注释)             │
├────────────────────────────────────┤
│  [上一步]  [下一步]  [重新开始]    │
├────────────────────────────────────┤
│  [看视频]    [讲解]    [练习]      │
└────────────────────────────────────┘
```

#### 练习模式 - 选择执红/执黑
```
┌────────────────────────────────────┐
│         棋盘区域                   │
├────────────────────────────────────┤
│    [执红练习]    [执黑练习]        │
├────────────────────────────────────┤
│  [看视频]    [讲解]    [练习]      │
└────────────────────────────────────┘
```

#### 练习模式 - 进行中
```
┌────────────────────────────────────┐
│         棋盘区域                   │
│      (可交互，AI自动回应)           │
├────────────────────────────────────┤
│  [悔棋]    [重新开始]              │
├────────────────────────────────────┤
│  [看视频]    [讲解]    [练习]      │
└────────────────────────────────────┘
```

### 2.4 底部三按钮固定

- `[看视频] [讲解] [练习]` 始终在最底部，不与其他按钮混排
- 当前模式对应的按钮高亮显示

### 2.5 文件修改

| 文件 | 修改内容 |
|-----|---------|
| `pages/game/game.wxml` | 新增四种状态的UI结构 |
| `pages/game/game.wxss` | 新增样式 |
| `pages/game/game.js` | mode状态管理，讲解/练习模式逻辑 |

## 三、实现顺序 （只修改到第二步）

```
1. opening 页面重构
   - 简化逻辑（去掉详情）
   - 新UI（左侧Tab + 右侧列表）
   ↓
2. game 页面练习模式
   - 实现执红/执黑选择
   - 复用现有走子验证逻辑
   ↓
3. game 页面讲解模式（暂不实现）
   - 棋盘显示
   - 上一步/下一步遍历
   - 讲解内容区（简介/注释切换）
   ↓
4. game 页面的视频模式（暂不实现）
   - 视频播放
```

---

## 四、game.js 状态设计

```javascript
data: {
  mode: 'video',           // 'video' | 'explain' | 'practice'
  practiceState: 'select',  // 'select' | 'playing'
  currentStep: 0,           // 讲解模式当前步数
  explainContent: '',      // 讲解内容
}

onLoad(options) {
  // 默认 mode: 'video'
  this.setData({ mode: 'video' })
}
```

---

## 五、注意事项

1. 练习模式复用现有 `onBeforeDrop` 走子验证逻辑
2. 讲解模式的上一步/下一步是遍历棋谱树，不触发AI回应
3. 悔棋必须悔两步（用户+AI各一步）
4. 练习模式执黑：AI自动走第一步（棋谱首步无分支）
5. 练习模式执红：用户走第一步，黑方有分支随机选择
