# 中国象棋·开局大师 - 设计方案

## 1. 设计目标

将现有的棋盘组件和规则引擎整合为一款专注开局练习的微信小程序。核心定位是"严格棋谱模式"，让用户通过反复练习熟记经典开局。

## 2. 页面结构（4 页核心 + 1 加载过渡）

| 页面 | 路径 | 职责 |
|------|------|------|
| 加载页 | `pages/loading/loading` | 首屏动画、本地数据预检、微信登录态检查 |
| 首页 | `pages/index/index` | 五大体系入口网格、用户信息、最近练习快捷入口 |
| 开局页 | `pages/opening/opening` | **变例列表 + 教学视频 + 逐变讲解 + 开始练习**（合并原分类页与详情页） |
| 对弈页 | `pages/game/game` | 严格棋谱模式对弈（已有基础，接入真实棋谱数据） |
| 我的 | `pages/profile/profile` | 用户信息、已购开局列表、设置 |

### 页面跳转流程

```
加载页 → 首页 → 开局页（含视频/逐变讲解） → 对弈页
                ↓
              我的页
```

## 3. 开局页（opening）布局设计

采用**上下分栏**方案：

```
┌─────────────────────────┐
│  < 返回    [体系名称]    │
├─────────────────────────┤
│  [变例 1]  ← 选中高亮   │
│  [变例 2]               │
│  [变例 3]  🔒           │
├─────────────────────────┤
│  ┌─────────────────┐    │
│  │   教学视频      │    │
│  └─────────────────┘    │
│  [逐变讲解]  [开始练习]  │
│  ─────────────────────  │
│  逐变讲解树（可展开）    │
│  ─────────────────────  │
│  变例简介文字            │
└─────────────────────────┘
```

### 交互规则

- 顶部显示当前体系名称（通过首页传入 `?category=zhongpao`）。
- 变例列表为垂直列表，每行显示名称、锁定图标。
- 点击变例卡片后：
  - 卡片高亮
  - 下方详情区刷新为该变例的视频、逐变讲解、简介
  - "开始练习"按钮根据 `locked` 状态变化：免费/已购则可用，锁定则显示"解锁"
- 默认选中第一个未锁定的变例。
- 逐变讲解分支点击后，弹出**半屏棋盘浮层**，预览该分支的下一步局面。

## 6. 对弈页（game）核心逻辑

### 6.1 页面参数

- `openingId`: 要练习的开局 ID
- `side`: 用户选择的先后手，`red` 或 `black`

### 6.2 严格棋谱模式规则

1. **加载棋谱**：从 `data/openings/{openingId}.json` 读取棋谱树。
2. **合法性检查**：用户走子先经 `xiangqi.js` 引擎验证，非法则禁止落子。
3. **棋谱匹配**：
   - 非棋谱走法 → 提示"非当前棋谱棋路，请再回去继续学习当前棋谱吧~"，不走棋，引擎回退。
   - 典型错误走法 → 提示 `errorComment`，允许落子，AI 按错误分支的子节点回应。
   - 正确棋谱走法 → 正常走子，AI 按主变（`children[0]`）回应。
4. **AI 回应**：不做任何计算，仅从棋谱树中取下一步。
5. **棋谱结束**：当前节点无子节点时，弹出"棋谱练习结束"，提供"重新练习"和"返回首页"。
6. **悔棋**：回退两步（用户一步 + AI 一步），沿 `parent` 链回退，不保留分支历史。
7. **中途退出**：直接返回，不保存对局。

## 7. 工具与脚本

### PGN 预编译脚本

- 位置：`scripts/parse-pgn.js`（Node.js，不打包进小程序）
- 输入：`docs/*.pgn`
- 输出：`data/openings/*.json`
- 职责：解析变着层级、提取注释、标记主变/错误分支、转换为引擎坐标。

## 8. 开发里程碑

| 阶段 | 目标 | 验证标准 |
|------|------|----------|
| **M1：骨架与数据** | 4 页空壳 + PGN 解析器 + 示例数据 | 首页 → 开局页 → 对弈页跳转正常；示例棋谱正确解析为 JSON |
| **M2：开局页闭环** | `opening` 页变例切换、视频区、逐变讲解预览 | 点击不同变例，详情区同步刷新；点击分支能预览局面 |
| **M3：对弈模式完善** | 严格棋谱模式接入真实棋谱、悔棋、胜负、错误提示 | 任意开局能完整练习；非棋谱步拦截；错误步提示并继续 |
| **M4：用户与付费** | 微信登录、锁定标记、支付解锁、设置页 | 能登录；锁态正确；支付流程跑通 |
